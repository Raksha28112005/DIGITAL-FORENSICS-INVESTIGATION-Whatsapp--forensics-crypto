How WhatsApp
Encrypts Your Data

WhatsApp uses AES-256-GCM encryption — the same standard used by banks, governments, and militaries worldwide. Here's exactly how it works.

AES-256 — The Algorithm
AES
FULL NAME

Advanced Encryption Standard. Created by the US government in 2001. Used worldwide as the gold standard for encryption.

256
KEY SIZE (BITS)

The key is 256 bits long — that is 32 bytes. There are 2²⁵⁶ possible keys. A supercomputer would take longer than the age of the universe to guess it.

GCM
MODE

Galois/Counter Mode. Not only encrypts the data but also creates a tag to verify the data was not tampered with during storage or transfer.

How WhatsApp Uses AES-256
1
You Send a Message

You type "Hey!" and press send in WhatsApp. Before it leaves your device, WhatsApp immediately begins the encryption process.

Plaintext: "Hey!"
2
Secret Key is Used

A unique 256-bit (32-byte) AES key is stored on your device. This key was generated when you first installed WhatsApp and is stored deep inside the phone at a protected location.

Key: cd788b1b4625f50d3fccdeac94e1ff638899733b77a224ff614918363901f044
3
IV (Initialization Vector) Added

A random 16-byte IV is generated for each encryption. Even if you send the same message twice, the encrypted output will be completely different each time — making it impossible to guess patterns.

IV: ea53ceae36ecab50bc331aeb62491625
4
Message is Encrypted → Saved as .crypt14

AES-256-GCM combines the key + IV + message to produce ciphertext. The database is saved on your phone as msgstore.db.crypt14

Ciphertext: 7f3a9c1d4e8b2f0a5c7d9e3f1a4b6c8d...
5
Forensic Investigator Decrypts

Our Python tool reads the key file, extracts the IV from the database header, and runs AES-256-GCM decryption to recover the original SQLite database — revealing all messages as evidence.

cipher = AES.new(key, AES.MODE_GCM, nonce=iv)
decrypted = cipher.decrypt(ciphertext)
# Result: valid SQLite database ✅
Key Cryptography Terms
🔑
Encryption Key

A secret 32-byte (256-bit) random number used to lock and unlock the database. Without this key, the crypt14 file is completely unreadable.

🎲
IV — Initialization Vector

A random 16-byte value added to encryption to ensure the same message always produces different ciphertext. Stored in the database header.

🏷️
GCM Authentication Tag

A 16-byte digital fingerprint attached to the encrypted data. If anyone modifies even one byte of the database, the tag becomes invalid — proving tampering.

🔏
SHA-256 Hash

A one-way fingerprint of a file. Used in our project for Chain of Custody — if the file changes even 1 bit, the hash changes completely, proving the evidence is authentic.

🗄️
SQLite Database

After decryption, the .crypt14 file becomes a standard SQLite database — like an Excel file for apps. Contains tables with all messages, contacts, and media references.

📱
.crypt14 Format

WhatsApp's proprietary encrypted backup format. The "14" means version 14 of their encryption. Contains a protobuf header, IV, ciphertext, and GCM authentication tag.

Why This Matters for Forensics
🔓
Legal Evidence

Decrypted messages with SHA-256 hashes can be submitted as verified digital evidence in legal proceedings.

🛡️
Tamper Detection

The GCM authentication tag proves the database has not been modified since it was created on the device.

⏱️
Timeline Analysis

Every message has a Unix timestamp. Investigators can reconstruct a complete communication timeline for any case.
